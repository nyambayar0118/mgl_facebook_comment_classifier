Монгол Facebook-ийн Spam Сэтгэгдэл Илрүүлэх Машин Сургалтын Төсөл

Author

Buyadaa

Published

January 1, 2025

Оршил

Интернэт орчинд төөрөгдүүлсэн, сурталчилгааны зорилготой болон хуурамч мэдээлэл агуулсан spam сэтгэгдлүүд нь олон нийтийн мэдээллийн хэрэгслүүдийн аюулгүй байдлыг алдагдуулах томоохон асуудлын нэг юм. Тухайлбал, Facebook орчинд хийгддэг автомат сэтгэгдлийн спам нь хэрэглэгчийн туршлагыг муутгахын зэрэгцээ хуурамч зар сурталчилгааг түгээх, залилангийн эрсдэлийг нэмэгдүүлдэг. Тиймээс Facebook-ийн спам сэтгэгдлийг машин сургалтын аргаар илрүүлэх нь ач холбогдол өндөртэй бөгөөд энэ төслийн үндсэн зорилго болно.

Энэхүү тайланд Монгол хэл дээр бичигдсэн Facebook сэтгэгдлүүдийг үндэслэн spam эсэхийг тодорхойлох машин сургалтын сайн чанартай загвар боловсруулах үе шат, хэрэглэсэн өгөгдөл, хийсэн цэвэрлэгээ, шинжилгээ болон сургасан дөрвөн төрөлт загварын тайлбар багтана.

1. Өгөгдөл цуглуулалт

Төслийн үр дүнгээс хамааран хамгийн чухал суурь алхам бол өгөгдлөө зөв, хангалттай хэмжээгээр бүрдүүлэх юм. Энэхүү төсөлд Facebook-ийн олон нийтийн хуудас, нийтлэлүүдийн сэтгэгдлүүдийг гараар болон автомат хэрэгслүүдээр дамжуулан татаж авсан. Өгөгдөл нь хоёр ангилалтай:

  • Spam сэтгэгдэл — сурталчилгааны, холбоосоор хандуулах, санхүүгийн залилан агуулсан, олон удаа давтагдсан автомат сэтгэгдлүүд.
  • Бутархай / энгийн сэтгэгдэл — хэвийн хэрэглэгчийн бичсэн, сэдвийн хүрээнд мэдээлэл агуулсан сэтгэгдлүүд.

Цуглуулсан өгөгдөлд: - сэтгэгдлийн текст, - бичсэн огноо, - хэрэглэгчийн ID (зөвхөн ялгалтын зорилгоор, загварт ашиглаагүй), - spam эсэхийг илэрхийлэх label гэх мэт талбарууд багтсан.

2. Өгөгдөл унших ба анхан шатны боловсруулалт

Татаж авсан өгөгдлийг төслийн орчинд импортлон унших үед сэтгэгдлүүдийн бүтэц, текстийн чанарт тулгуурлан дараах асуудлууд илэрсэн:

  • давхардсан сэтгэгдэл олон байсан,
  • URL, emoji, тусгай тэмдэгтүүд хэт их агуулагдсан,
  • зарим урт сэтгэгдэл нь автоматаар нэг мөрт багтсан,
  • хоосон зай болон формат алдагдсан хэсгүүдтэй байсан.

Анхан шатны боловсруулалтад: - давхардлыг устгах, - эмоджи болон илүү тэмдэгтүүдийг арилгах, - текстийг жижиг үсэг рүү хөрвүүлэх, - хоосон мөр, утгагүй тэмдэгтүүдийг устгах зэрэг үйлдлүүд хийгдсэн.

3. Өгөгдөл цэвэрлэх

Спам илрүүлэх зэрэг текстэн өгөгдлийн машин сургалтын ажилд өгөгдөл цэвэрлэх нь маш чухал. Энэ төсөлд дараах техникүүд ашиглав:

3.1 Stopwords устгах

Монгол хэлний байна, бол, гээд, юм зэрэг утга агуулаагүй ч давтамж өндөр үгс нь загварт сөргөөр нөлөөлдөг тул stopword жагсаалтыг ашиглан устгав.

3.2 Tokenization

Сэтгэгдлүүдийг: - Unigram буюу нэг нэгээр нь, - Bigram буюу хоёр үгийн дараалал болгон хуваах токенчлолын техникийг ашигласан.

3.3 Стемминг/Лемматизаци

Монгол хэлний стемминг төдийлөн төгөлдөржөөгүй тул төслийн хүрээнд үндсэндээ үеэр таслах, үгийн үндэс хэлбэрийг үлдээх хөнгөн цэвэрлэгээ хийсэн.

4. Өгөгдөл хуваах

Загварын гүйцэтгэлийг үнэлэхийн тулд өгөгдлийг: - Train set — 80%, - Test set — 20% хувиар хуваав. Хуваахдаа ангиллын тэнцвэрт байдлыг хадгалахын тулд stratified split ашиглав.

5. Өгөгдлийн тархалтын дүрслэл

Тархалтын графикууд нь spam болон normal сэтгэгдлүүдийн үгийн давтамж, урт, хамгийн түгээмэл үгсийн ялгааг ажиглахад тусалсан. Үүнд: - үгийн урт, - нийт token-ийн тоо, - spam ангилалд хамгийн их давтамжтай үгс, - normal ангилалд хамгийн их давтамжтай үгсмэт хэмжигдэхүүнүүд багтсан.

Дүрслэлүүдээс харахад spam сэтгэгдэлд: - мөнгө, зар, борлуулалтын шинжтэй үгс давамгайлсан, - URL болон emoji илүү байсан, - үгийн давтамж жигд бус хуваарилалттай гарсан.

6. Машин сургалтын загварууд

Энэхүү төсөлд дөрвөн загвар хэрэглэв. Текстэн өгөгдлийн онцлогоос хамааран Naive Bayes болон Decision Tree алгоритмууд тохиромжтой гэж үзсэн.

6.1 Naive Bayes (Unigram)

Unigram Naive Bayes нь сэтгэгдлийг тус тусын үгсэд салгасан дараа үг бүрийн нөхцөлт магадлал дээр үндэслэн ангилдаг. Давуу тал нь: - хурдан, - өгөгдөл багатай үед сайн ажилладаг, - текстанд үр дүнтэй.

Туршилтын үр дүнгээс харахад Unigram загвар нь бусадтай харьцуулахад илүү тогтвортой, baseline амжилттай гарсан.

6.2 Naive Bayes (Bigram)

Bigram Naive Bayes нь хоёр үгийн дараалал ашигладаг тул илүү нарийн контекст ойлгодог. Жишээлбэл: - “үнэгүй ав” → spam давтамж өндөр - “сайн байна” → normal сэтгэгдэлд түгээмэл

Bigram нь илүү сайн контекст барьдаг ч үгийн сан хэт өсөх тул өгөгдөл цөөтэй үед overfitting үүсэх магадлалтай. Туршилтаар харьцангуй сайн гарсан ч Unigram-аас хэт давахгүй байв.

6.3 Decision Tree Classifier

Decision Tree нь багана бүрээр нөхцөл шалган мод хэлбэртэй шийдвэр гаргадаг. Давуу тал: - тайлбарлахад хялбар, - дүрэм суурьтай, - олон төрлийн feature-т сайн.

Гэхдээ текстэн материалд dimensionality маш өндөр байдаг тул мод хэт томорч, гүйцэтгэл зарим үед буурсан.

6.4 Unified Model

Unified model нь дээрх гурван загварыг нэгтгэн ensemble хэлбэрээр дундажлах/жигнэх аргыг ашигласан. Энэхүү загварын зорилго: - загваруудын сул талыг нөхөх, - илүү тогтвортой, дундаж өндөр нарийвчлалтай систем бий болгох.

Үр дүнгээрээ unified model хамгийн өндөр F1-score үзүүлсэн.

7. Үр дүнгийн дүн шинжилгээ

Загваруудын гүйцэтгэлийг дараах хэмжигдэхүүнүүдээр хэмжсэн: - Accuracy - Precision - Recall - F1-score

Ерөнхий ажиглалтаас харахад: - Unigram NB → өндөр recall - Bigram NB → текстийн утга барих чадвар илүү - Decision Tree → тайлбарлахад сайн ч dimensionality-н улмаас харьцангуй сул - Unified Model → хамгийн тэнцвэртэй, өндөр гүйцэтгэлтэй

8. Дүгнэлт

Энэхүү төслийн хүрээнд Монгол хэл дээрх Facebook сэтгэгдлүүдийн spam илрүүлэх системийг боловсруулж, өгөгдөл цуглуулах, боловсруулалт хийх, шинжилгээ хийх, хэд хэдэн машин сургалтын загвар турших ажлуудыг амжилттай хийж гүйцэтгэлээ.

Unified Model нь хамгийн сайн гүйцэтгэлтэй гарсан бөгөөд цаашид: - илүү том өгөгдлийн сан бүрдүүлэх, - deep learning (LSTM, BERT) турших, - real-time spam detection системд холбох боломжтой юм.

Энэхүү тайлан нь кодын хэсгүүдийг оруулахгүй зөвхөн судалгааны үндсэн агуулгыг багтаасан болно.
